## 1.2 COMMUNITY POSTS CONTROLLER
Base Path: /api/community/posts
Auth: JWT + RolesGuard

ENDPOINTS:

1. GET /api/community/posts
   - List community posts with filtering
   - Query Params:
     * courseId?: number (filter)
     * postType?: PostType (filter)
     * sortBy?: 'recent' | 'popular' | 'most_comments' (default: recent)
     * page?: number (default: 1)
     * limit?: number (default: 20)
   - Response: { data: CommunityPost[], meta: { total, page, limit, totalPages } }

2. POST /api/community/posts
   - Create new post
   - Body: CreatePostDto { title, content, courseId, postType? }
   - Response: CommunityPost

3. GET /api/community/posts/:id
   - Get post with comments
   - Params: id (post ID)
   - Query: page?, limit? (for comments pagination)
   - Response: { post, comments: { data[], meta }, reactions }
   - Note: Increments view count

4. PUT /api/community/posts/:id
   - Update post (owner only)
   - Params: id
   - Body: UpdatePostDto { title?, content?, postType? }
   - Response: CommunityPost

5. DELETE /api/community/posts/:id
   - Delete post (owner or admin)
   - Params: id
   - Response: 204 No Content

6. POST /api/community/posts/:id/comment
   - Add comment to post
   - Params: id (post ID)
   - Body: CreateCommentDto { commentText, parentCommentId? }
   - Response: CommunityComment
   - Note: Fails if post is locked

7. POST /api/community/posts/:id/react
   - Toggle reaction on post
   - Params: id (post ID)
   - Body: CreateReactionDto { reactionType }
   - Response: { action: 'added'|'removed', reactionType }
   - Note: Toggle behavior - same reaction removes it

8. PATCH /api/community/posts/:id/pin
   - Toggle pin post (instructor/admin only)
   - Params: id
   - Roles: INSTRUCTOR, ADMIN, IT_ADMIN
   - Response: CommunityPost

9. PATCH /api/community/posts/:id/lock
   - Toggle lock post (instructor/admin only)
   - Params: id
   - Roles: INSTRUCTOR, ADMIN, IT_ADMIN
   - Response: CommunityPost

## 1.5 COMMUNITY ENTITIES

### CommunityPost Entity
Columns:
- id: number (PK, bigint)
- courseId: number (FK to Course)
- userId: number (FK to User - author)
- title: string (varchar 255)
- content: text
- postType: enum (DISCUSSION, QUESTION, ANNOUNCEMENT, RESOURCE)
- isPinned: boolean (tinyint, default 0)
- isLocked: boolean (tinyint, default 0)
- viewCount: int (default 0)
- upvoteCount: int (default 0)
- replyCount: int (default 0)
- createdAt: timestamp
- updatedAt: timestamp

Relations:
- author: User (ManyToOne, eager)
- course: Course (ManyToOne, eager)
- comments: CommunityComment[] (OneToMany)
- reactions: CommunityReaction[] (OneToMany)

### CommunityComment Entity
Columns:
- id: number (PK, bigint)
- postId: number (FK to CommunityPost)
- userId: number (FK to User - author)
- commentText: text
- parentCommentId?: number (nullable, supports nested replies)
- upvoteCount: int (default 0)
- createdAt: timestamp
- updatedAt: timestamp

Relations:
- author: User (ManyToOne, eager)
- post: CommunityPost (ManyToOne, cascade delete)
- parentComment: CommunityComment (ManyToOne, nullable)

### CommunityReaction Entity
Columns:
- id: number (PK, bigint)
- postId: number (FK to CommunityPost)
- userId: number (FK to User)
- reactionType: enum (LIKE, HELPFUL, INSIGHTFUL, THANKS)
- createdAt: timestamp

Relations:
- user: User (ManyToOne, eager)
- post: CommunityPost (ManyToOne, cascade delete)

Unique Constraint: [postId, userId, reactionType]

### ForumCategory Entity
Columns:
- id: number (PK, bigint)
- courseId: number (FK to Course)
- name: string (varchar 255, nullable)
- description: text (nullable)
- orderIndex: int (nullable)
- createdAt: timestamp

Relations:
- course: Course (ManyToOne, eager)

## 2.2 DISCUSSIONS CONTROLLER
Base Path: /api/discussions
Auth: JWT + RolesGuard

ENDPOINTS:

1. GET /api/discussions
   - List discussion threads
   - Query: courseId?: number, page?: number (default 1), limit?: number (default 20)
   - Response: { data: CourseChatThread[], meta: { total, page, limit, totalPages } }
   - Note: Pinned threads appear first

2. POST /api/discussions
   - Create discussion thread
   - Body: CreateThreadDto { courseId, title, description? }
   - Response: CourseChatThread

3. GET /api/discussions/:id
   - Get thread with replies (paginated)
   - Params: id (thread ID)
   - Query: page?, limit? (for replies)
   - Response: { thread, replies: { data[], meta } }
   - Note: Increments view count, answers appear first

4. PUT /api/discussions/:id
   - Update thread (author, instructors, admins)
   - Params: id
   - Body: UpdateThreadDto { title?, description? }
   - Response: CourseChatThread

5. DELETE /api/discussions/:id
   - Delete thread (instructors/admins only)
   - Roles: INSTRUCTOR, ADMIN, IT_ADMIN
   - Params: id
   - Response: 204 No Content

6. POST /api/discussions/:id/reply
   - Post reply to thread
   - Params: id (thread ID)
   - Body: CreateReplyDto { messageText, parentMessageId? }
   - Response: ChatMessage
   - Note: Fails if thread is locked

7. PATCH /api/discussions/:id/pin
   - Toggle pin thread
   - Roles: INSTRUCTOR, TA, ADMIN, IT_ADMIN
   - Params: id
   - Response: CourseChatThread

8. PATCH /api/discussions/:id/lock
   - Toggle lock thread (prevents new replies)
   - Roles: INSTRUCTOR, TA, ADMIN, IT_ADMIN
   - Params: id
   - Response: CourseChatThread

9. PATCH /api/discussions/replies/:replyId/mark-answer
   - Mark/unmark reply as accepted answer
   - Roles: INSTRUCTOR, TA, ADMIN, IT_ADMIN
   - Params: replyId (message ID)
   - Response: ChatMessage
   - Note: Answers appear first in replies

10. PATCH /api/discussions/replies/:replyId/endorse
    - Toggle endorse reply (instructor approval)
    - Roles: INSTRUCTOR, TA, ADMIN, IT_ADMIN
    - Params: replyId (message ID)
    - Response: ChatMessage

## 2.3 DISCUSSION ENTITIES

### CourseChatThread Entity
Columns:
- id: number (PK, bigint)
- courseId: number (FK to Course)
- createdBy: number (FK to User)
- title: varchar (255)
- description: text (nullable)
- isPinned: boolean (tinyint, default 0)
- isLocked: boolean (tinyint, default 0)
- viewCount: int (default 0)
- replyCount: int (default 0)
- createdAt: timestamp
- updatedAt: timestamp

Relations:
- messages: ChatMessage[] (OneToMany)
- course: Course (ManyToOne, cascade delete)
- creator: User (ManyToOne, cascade delete)

### ChatMessage Entity
Columns:
- id: number (PK, bigint)
- threadId: number (FK to CourseChatThread)
- userId: number (FK to User)
- messageText: text
- parentMessageId?: number (nullable, for nested replies)
- isAnswer: boolean (tinyint, default 0)
- isEndorsed: boolean (tinyint, default 0)
- endorsedBy?: number (nullable, FK to User who endorsed)
- upvoteCount: int (default 0)
- createdAt: timestamp
- updatedAt: timestamp

Relations:
- thread: CourseChatThread (ManyToOne)
- user: User (ManyToOne, cascade delete)
- endorser: User (ManyToOne, nullable, set null on delete)
- parentMessage: ChatMessage (ManyToOne, nullable, set null on delete)

# 5. AUTHENTICATION & AUTHORIZATION

### Guards
- JwtAuthGuard: All endpoints require JWT token
- RolesGuard: Used for role-based access

### Roles
Available roles: INSTRUCTOR, ADMIN, IT_ADMIN, TA, STUDENT, etc.

### Admin-Only Operations
- Create forum categories
- Update forum categories
- Delete forum categories
- Delete discussion threads (instructor+)

### Instructor-Only Operations
- Pin/lock posts
- Pin/lock threads
- Mark thread replies as answer
- Endorse replies
- Update threads (+ thread author)
- Delete threads
